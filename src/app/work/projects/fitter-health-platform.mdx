## Overview

Fitter Health is A preventive health-tech platform I built from the ground up. Not a feature bolted onto someone else's codebase, but the whole product: frontend, backend, database, cloud infrastructure, and data layer, owned from the first commit through to production for a Spain-based team.

## What I built

**Frontend.** A modern Next.js interface for the platform's product modules.

**Backend.** Django and Django REST Framework APIs, with every contract and workflow designed for reliability and maintainability.

**Cloud.** The full AWS stack: ECS, RDS, S3, SNS, SQS, Redshift, API Gateway, IAM, and VPC. Containerized with Docker and production-secured.

**Async and data.** Celery and Redis background processing, a notification and file-handling architecture, and a data layer ready for analytics and data-driven features.

**Production quality.** Automated tests, clean migrations, and database-level integrity constraints throughout.

## The point

This is the kind of work I do: complete, production-grade software products, front page to database, not just a website.

**Stack:** Next.js · Django · DRF · PostgreSQL · Celery · Redis · Docker · AWS (ECS, RDS, S3, SNS, SQS, Redshift, API Gateway, IAM, VPC)
